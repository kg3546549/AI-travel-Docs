## AI 역할 정의

AI는 **구조/동선 설계자** 역할만 담당. 실제 장소 데이터와 가격은 외부 API가 채운다.
AI가 직접 좌표를 출력하면 할루시네이션이 발생하므로 좌표는 반드시 Places API에서 조회한다.

```
[AI 담당]              [API 담당]
지역 순서 결정          Google Places API → 실제 장소 데이터 + 좌표
동선 최적화             Amadeus API → 실제 항공권 가격
미정 조건 추천          Naver Local API → 한국 내 장소 보완
예산 배분
```

---

## AI 검색 최적화 전략

1. **검색용 키워드 분리:** 장소명 + 검색에 최적화된 키워드를 별도 필드로 출력
   - 예: `"name": "도쿄 타워"`, `"search_query": "Tokyo Tower, Minato City, Tokyo"`
2. **지역 컨텍스트 포함:** 도시 영문명/현지명을 검색 키워드에 반드시 포함
3. **카테고리 힌트 제공:** 식당/카페/관광지 명시 → Google API `type` 필터링에 활용

## 2단계 AI 호출 구조

### 1차 호출 - 요약 후보 3~4개

**System Prompt**
```
당신은 여행 일정 전문가입니다.
사용자 조건을 받아 스타일이 다른 여행 계획 후보 3~4개를 JSON으로 반환합니다.
후보는 키워드와 주요 장소 요약만 포함하며, 상세 일정은 포함하지 않습니다.
반드시 아래 JSON 스키마만 반환하세요.
```

**응답 스키마**
```json
{
  "candidates": [
    {
      "id": 1,
      "keyword": "맛집 집중형",
      "summary": "현지 유명 식당 중심, 지역 이동 최소화",
      "key_spots": ["시부야", "니시오기쿠보", "나카메구로"],
      "estimated_budget": 800000
    },
    {
      "id": 2,
      "keyword": "관광지 위주",
      "summary": "도쿄 대표 명소 집중 탐방",
      "key_spots": ["아사쿠사", "황거", "오다이바"],
      "estimated_budget": 750000
    }
  ]
}
```

### 2차 호출 - 선택된 후보의 상세 일정

**User Prompt**
```
목적지: 도쿄
기간: 3박 4일 (2026-04-01 ~ 2026-04-04)
인원: 2명 (커플)
예산: 80만원 (항공 포함)
스타일: 맛집 집중형
출발지: 서울
이동수단: 대중교통
일정강도: 보통
추가 요청: 해리포터 스튜디오 꼭 가고 싶어요
```

**응답 스키마**
```json
{
  "title": "도쿄 3박 4일 - 맛집 집중형",
  "budget_breakdown": {
    "flight": 280000,
    "accommodation": 240000,
    "food": 180000,
    "activities": 100000
  },
  "days": [
    {
      "day": 1,
      "theme": "시부야·하라주쿠",
      "spots": [
        {
          "order": 1,
          "name": "이치란 라멘 시부야점",
          "search_query": "Ichiran Ramen Shibuya, Tokyo",
          "category": "restaurant",
          "duration_min": 45,
          "estimated_cost": 15000,
          "notes": "점심"
        }
      ]
    }
  ]
}
```

## 전체 데이터 흐름

```
사용자 입력
    ↓
1차 AI 호출 → 후보 3~4개 (keyword + summary + key_spots)
    ↓
사용자가 후보 선택
    ↓
2차 AI 호출 → 상세 일정 JSON (장소명 + search_query + 카테고리)
    ↓
각 spot.search_query → Google Places API → 좌표 + place_id 추가
    ↓
DB 저장 (trips + itinerary_days + spots)
    ↓
지도 렌더링 + 일정표 표시
```
